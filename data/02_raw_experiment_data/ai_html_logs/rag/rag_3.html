Solución (Instrucciones de implementación)

Lógica del sistema: El dispositivo medirá la aceleración constante. Si el movimiento supera un umbral mínimo, se activará un sonido de batería. Se utilizan dos niveles de alerta para diferenciar un sismo menor de un terremoto.

Pseudocódigo de la solución:

START
  // Definición de variables de umbral sísmico
  Let umbralLeve = 15
  Let umbralFuerte = 40

  MAIN LOOP:
    // Capturar la fuerza del movimiento actual
    Let intensidad = get property 'acceleration' from [Motion 1]

    IF intensidad > umbralFuerte:
      // Alerta Magnitud Alta: Sonido constante y fuerte
      Execute play on [Musical Drum 1] with drumType 'bassDrum' and intensity 'high'
      Wait 100 milliseconds
    ELSE IF intensidad > umbralLeve:
      // Alerta Magnitud Baja: Sonido intermitente
      Execute play on [Musical Drum 1] with drumType 'snare' and intensity 'low'
      Wait 300 milliseconds
    ELSE:
      // Estado de reposo o silencio sísmico
      Execute stop on [Musical Drum 1]
    END IF

    Wait 50 milliseconds
  END MAIN LOOP
END

Actividad: Sismógrafo de Bolsillo

1. Encabezado y Metadatos

  • Título de la actividad: Sismógrafo de Bolsillo: Escuchando la Tierra.
  • Edad objetivo: 12-13 años (Séptimo Básico).
  • Nivel de dificultad: 2 (Intermedio).
  • Temas de Computación: Sensores y Entrada (Input), Condicionales y Lógica, Variables de datos (Umbrales).
  • Temas Generales: Geografía, Riesgos Naturales, Relación Ser Humano-Medio Ambiente.
  • Etiquetas: Sensor de movimiento, Batería musical, Riesgos, Sismos, Geografía Física.

2. Introducción (El "Gancho")

¿Sabías que vivimos en uno de los planetas más activos del sistema solar? La Tierra siempre se está moviendo bajo nuestros pies, aunque no siempre lo sintamos. En Chile y otras regiones del "Cinturón de Fuego del Pacífico", entender estos movimientos es vital para nuestra seguridad. ¿Cómo podemos convertir esas vibraciones invisibles en sonidos y señales que podamos analizar? ¡Hoy construirás tu propio sensor sísmico para alertar a la población!

3. Objetivos Educacionales

  • Comprender el concepto de riesgo natural y cómo la tecnología ayuda a la prevención.
  • Aprender a utilizar un sensor de movimiento para medir la intensidad de una fuerza física.
  • Aplicar lógica condicional para categorizar datos (sismo leve vs. sismo fuerte).

4. Dispositivos a utilizar

  • Entrada: Sensor de movimiento (Motion Sensor).
  • Salida: Batería (Musical Drum).

5. Solución (Instrucciones de Programación)

Contexto narrativo: El dispositivo debe ser colocado sobre una superficie plana (el escritorio). Los estudiantes deben simular diferentes intensidades de sismos golpeando suave o fuertemente la mesa. El programa debe distinguir entre un movimiento ambiental normal y un evento sísmico.

Lógica del Pseudocódigo: El programa leerá constantemente la aceleración del sensor. Si el valor supera un "umbral de seguridad", activará un sonido de tambor. A mayor movimiento, el sonido será más intenso o frecuente.

Pseudocódigo (Estilo pseudocode.s.txt):

START
  // Configuración inicial de sensibilidad
  Let umbralLeve = 15
  Let umbralFuerte = 40

  MAIN LOOP:
    // Leer la intensidad del movimiento actual
    Let intensidad = get property 'acceleration' from [Motion 1]

    IF intensidad > umbralFuerte:
      // Sismo de gran magnitud: Sonido de bombo fuerte
      Execute play on [Musical Drum 1] with drumType 'bassDrum' and intensity 'high'
      Wait 100 milliseconds
    ELSE IF intensidad > umbralLeve:
      // Sismo menor: Sonido de caja ligero
      Execute play on [Musical Drum 1] with drumType 'snare' and intensity 'low'
      Wait 300 milliseconds
    ELSE:
      // Estado de calma: No hay sonido
      Execute stop on [Musical Drum 1]
    END IF

    Wait 50 milliseconds
  END MAIN LOOP
END

Estructura principal: Todo el código debe ir dentro de un bucle "MAIN LOOP" (repetir para siempre) para que el sensor esté siempre "escuchando" el movimiento de la tierra. Los bloques IF/ELSE IF se encuentran en la categoría "Lógica" y los valores del sensor en "Componentes".

6. Reflexión y Extensión

Preguntas de reflexión:

  • ¿Por qué es importante establecer un "umbral" (un valor mínimo) en el código? ¿Qué pasaría si el tambor sonara con el más mínimo movimiento del aire?
  • ¿Cómo crees que los geógrafos utilizan estos datos para diseñar mapas de riesgo en las ciudades?

  1. El Centinela del Altiplano Los alumnos deben programar un dispositivo que proteja los ecosistemas de altura. Usando el sensor de inclinación, el sistema detectará si el terreno sufre una erosión o deslizamiento; si el ángulo es peligroso, la matriz de leds mostrará una alerta visual y la lámpara cambiará a color rojo para avisar a las comunidades locales sobre el riesgo geográfico.

Mediante el sensor de movimiento y la batería (musical drum), el sistema responderá a las vibraciones del pupitre: cuanto más fuerte sea el movimiento detectado, más intensos y rápidos serán los golpes del tambor, permitiendo "escuchar" la magnitud de un movimiento telúrico.

  4. La Brújula de los Asentamientos Los estudiantes programarán una guía de navegación para antiguos exploradores. Usando la brújula y el teclado musical, el dispositivo emitirá una nota diferente para cada punto cardinal (Norte, Sur, Este, Oeste). El desafío es seguir una ruta "musical" para encontrar el lugar ideal para un asentamiento humano basado en la orientación y los recursos naturales.

  5. Semáforo de Contaminación Acústica Relacionando la geografía urbana con el impacto ambiental, los alumnos usarán el sensor de ruido y la lámpara. La lámpara brillará en verde en zonas silenciosas (parques), naranja en zonas residenciales y parpadeará en rojo en zonas de alto tráfico o industria, visualizando cómo las actividades humanas transforman el espacio geográfico.

Actividad: El Faro de la Niebla Eterna

Esta actividad está diseñada para estudiantes de 15 años (2º Medio), integrando conceptos de lógica condicional y eventos para resolver un problema de seguridad marítima mediante el uso de ProtoObject.

1. Encabezado y Metadatos

  • Título de la Actividad: El Faro de la Niebla Eterna.
  • Nivel: 2 (Intermedio).
  • Descripción corta: Salva a un barco perdido en la tormenta programando un faro inteligente que coordina señales visuales y sonoras mediante un código de emergencia.
  • Temas de Computación: Eventos (Pulsación), Lógica Condicional (IF/ELSE), Bucles (Repetición), Salidas de hardware (Luz y Sonido).
  • Temas Generales: Seguridad Marítima, Comunicación (Código Morse), Resolución de problemas.
  • Etiquetas: #Eventos, #Lógica, #Seguridad, #Aventura, #Hardware.

2. Introducción (El "Gancho")

¡Emergencia en la costa! Un buque de carga se ha quedado sin radar en medio de una niebla espesa y se dirige peligrosamente hacia los acantilados de la Isla de las Sombras. El sistema automático del faro se ha averiado. Como ingenieros de guardia, vuestra misión es programar un sistema de emergencia manual. El faro debe emitir una señal visual potente y una sirena de advertencia solo cuando activéis el sensor táctil, permitiendo guiar al capitán de forma segura hacia el puerto. ¿Podréis evitar el naufragio?

3. Componentes de Hardware

  • Botón Táctil 1: Actúa como el interruptor de emergencia manual del faro.
  • Lámpara 1: Representa la luz potente del faro (debe ser visible en la niebla).
  • Reproductor de Audio 1: Emite la sirena de niebla para guiar al barco por sonido.

4. Guía Paso a Paso

  1. Preparación del entorno: Conecta el Botón Táctil, la Lámpara y el Reproductor de Audio en la interfaz de ProtoObject.
  2. Configuración del Audio: Selecciona un sonido de "Sirena" o "Alarma" en el Reproductor de Audio.
  3. Lógica de Control: Debemos crear un bucle que revise constantemente si el botón está siendo presionado.
  4. Estado de Alerta: Si el botón se presiona, la lámpara debe parpadear (encenderse y apagarse rápidamente) para llamar la atención, y el audio debe sonar.
  5. Estado de Reposo: Si el botón no se presiona, todo debe permanecer apagado para ahorrar energía.

5. Pseudocódigo

Este código sigue la sintaxis de los modelos de referencia para implementar la lógica del faro.

START
  MAIN LOOP:
    // Verificamos si el capitán está solicitando ayuda mediante el botón
    IF get property 'isTouched' from [Botón Táctil 1] IS equal to true THEN
      
      // Iniciamos secuencia de señalización
      Execute play on [Reproductor de Audio 1] with sound 'fog_horn.mp3'
      
      // Parpadeo de emergencia (Repetir 3 veces la señal visual)
      REPEAT 3 TIMES:
        Set color of [Lámpara 1] to hex value #FFFFFF // Blanco intenso
        Wait 300 milliseconds
        Execute turnOff on [Lámpara 1]
        Wait 200 milliseconds
      END REPEAT
      
    ELSE
      // Si no hay nadie tocando el sensor, el faro descansa
      Execute turnOff on [Lámpara 1]
      Execute stop on [Reproductor de Audio 1]
    END IF
  END MAIN LOOP
END
