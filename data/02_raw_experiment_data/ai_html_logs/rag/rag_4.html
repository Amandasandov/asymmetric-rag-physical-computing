2. Introducción (El "Gancho")

¿Alguna vez has intentado dibujar una parábola perfecta solo con tus brazos? En matemáticas, el número que acompaña a la $x^2$ (el coeficiente $a$) decide si nuestra parábola es "flaquita" y alta o "gordita" y baja. ¡Hoy te convertirás en ese coeficiente! Al inclinar tu teléfono, verás cómo la curva reacciona a tus movimientos, ayudándote a visualizar qué sucede cuando $a$ aumenta o disminuye.

3. Materiales Necesarios

  • Hardware (Smartphone con Protobject):
    • Entrada: Sensor de Inclinación (Inclination).
    • Salida: Matriz de LED (LED Matrix) y Reproductor de Audio (Audio Player).

4. Instrucciones Paso a Paso

  1. Preparar el sensor: Identifica que usaremos el eje de inclinación (pitch) para capturar el ángulo.
  2. Definir la función: Nuestra función base será $f(x) = ax^2$.
  3. Mapear el movimiento: Cuando el dispositivo esté horizontal (0°), $a$ será pequeño (parábola abierta). Al inclinarlo hacia la vertical (90°), $a$ será grande (parábola cerrada).
  4. Feedback Visual: La Matriz de LED dibujará la forma resultante.
  5. Feedback Sonoro: Si la parábola es muy estrecha, el Reproductor de Audio emitirá un sonido de "tensión" o agudo.

5. El Desafío de Programación

Instrucciones de Implementación para la Extensión

1. El Problema Real

En las zonas escolares de Chile, la velocidad máxima es de $30 \text{ km/h}$. Los alumnos deben modificar su odómetro para que funcione como un radar. Si el tiempo que transcurre entre una detección y otra es menor a un umbral de seguridad, el sistema debe activar una alarma visual.

2. Modificación del Montaje

  • Añade la Lámpara en un lugar visible.
  • Define una distancia fija entre marcas (ejemplo: $1$ metro).

3. Lógica de Programación Avanzada (Pseudocódigo)

Para esta extensión, utilizaremos la función de tiempo del sistema para medir qué tan rápido pasa el móvil entre la marca A y la marca B.

START
  // Variables de control
  Let distanciaTotal = 0
  Let tiempoReferencia = 0
  Let limiteSeguridad = 1500 // 1.5 segundos entre marcas (umbral de velocidad)

  MAIN LOOP:
    IF get property 'detected' from [Motion 1] is equal to TRUE THEN
      
      // Calculamos cuánto tiempo pasó desde la última detección
      Let tiempoActual = get property 'currentTime' from [System]
      Let intervalo = tiempoActual - tiempoReferencia
      
      // 1. Actualizar odómetro
      Let distanciaTotal = distanciaTotal + 1
      Draw on [LED Matrix 1] with number 'distanciaTotal'
      
      // 2. Lógica de Radar (Física Aplicada)
      // Si el intervalo es menor al límite, significa que va muy rápido
      IF intervalo < limiteSeguridad THEN
        Set color of [Lamp 1] to hex value #FF0000 // Rojo: Exceso de velocidad
      ELSE
        Set color of [Lamp 1] to hex value #00FF00 // Verde: Velocidad segura
      END IF
      
      // Actualizamos el tiempo de referencia para la próxima marca
      Let tiempoReferencia = tiempoActual
      
      Wait 500 milliseconds
    END IF
  END MAIN LOOP
END

Instrucciones Detalladas de Implementación

1. Introducción (El "Hook")

¿Alguna vez has notado que al medir con una regla y un cronómetro al mismo tiempo, los datos nunca parecen cuadrar perfectamente? Esto ocurre por el "error experimental". En la industria, desde el control de velocidad en autopistas hasta las cintas transportadoras de minería en Chile, se utilizan sensores que no tocan el objeto. Hoy, programarás tu propio sistema de telemetría para medir la velocidad de un objeto en movimiento rectilíneo sin usar una sola cinta métrica.

2. El Desafío Científico

Debes construir un dispositivo que registre cuántas "marcas de posición" cruza un móvil. Tu programa debe transformar una señal física (presencia) en un dato digital (conteo). Al finalizar, usarás la relación: $$V = \frac{n \cdot k}{\Delta t}$$ Donde $n$ es el número en tu Matriz LED, $k$ es la distancia entre marcas y $\Delta t$ el tiempo total.

3. Instrucciones de Montaje Paso a Paso

  1. Preparación de la Pista:
    • Extiende una pista recta de al menos 2 metros.
    • Coloca marcas visuales (o pequeños obstáculos) cada 20 cm.
  2. Configuración del Hardware:
    • Fija el Sensor de Movimiento en un soporte vertical al costado de la pista.
    • Asegúrate de que el sensor apunte perpendicularmente a la trayectoria del móvil.
  3. Configuración del Software (Protobject):
    • Conecta tu dispositivo móvil o computadora a los componentes Motion y LED Matrix.
    • Crea una variable llamada distanciaAcumulada para almacenar los datos.

4. Lógica de Programación (Pseudocódigo)

Este código sigue la sintaxis de pseudocode.s.txt. Es fundamental respetar el bloque de espera para que el sensor no cuente el mismo objeto múltiples veces mientras pasa.

START
  // Inicialización: Preparamos el sistema para el experimento
  Let distanciaAcumulada = 0
  Execute clear on [LED Matrix 1]
  Draw on [LED Matrix 1] with number 'distanciaAcumulada'

  MAIN LOOP:
    // El sensor monitorea el paso del móvil continuamente
    IF get property 'detected' from [Motion 1] is equal to TRUE THEN
      
      // Lógica de incremento: añadimos una unidad de medida
      Let distanciaAcumulada = distanciaAcumulada + 1
      
      // Feedback inmediato: mostramos el progreso en la matriz
      Draw on [LED Matrix 1] with number 'distanciaAcumulada'
      
      // IMPORTANTE: Pausa de 600ms para permitir que el móvil pase
      // Esto evita que una sola detección sume varios puntos
      Wait 600 milliseconds
      
    END IF
  END MAIN LOOP
END

5. Procedimiento Experimental para el Alumno

  1. Pon el contador en 0.
  2. Prepara el cronómetro.
  3. Suelta el móvil desde el reposo (o con velocidad constante).
  4. Detén el cronómetro cuando el móvil pase la última marca.
  5. Registra el valor $n$ que quedó en la Matriz LED.
